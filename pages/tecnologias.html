Tecnologias

Usadas em projetos do Portfólio

HTML5

HTML (Hypertext Markup Language) é a linguagem de marcação padrão usada para criar páginas web. Ele define a estrutura básica e os elementos de uma página, como cabeçalhos, parágrafos, listas, links, imagens, entre outros. Sua 5ª versão traz uma série de recursos novos e avançados para criar conteúdo multimídia e interativo na web.

Prós

  • Linguagem de marcação simples e fácil de aprender
  • É suportado por todos os principais navegadores e plataformas
  • Oferece APIs poderosas para interatividade, como geolocalização e armazenamento local

Limitações

  • Pode ser limitado em termos de design e interatividade sem o uso de CSS e JavaScript
  • Não é uma linguagem de programação completa, o que limita sua capacidade de lidar com lógica complexa
  • Possíveis inconsistências na renderização entre diferentes navegadores e dispositivos

CSS3

CSS (Cascading Style Sheets) é uma linguagem de estilo usada para descrever a apresentação de um documento HTML ou XML, incluindo cores, layout e fontes. Ele permite separar o conteúdo da apresentação visual.

Prós

  • Criação de layouts flexíveis e responsivos para páginas web
  • Facilita a padronização e a consistência visual em um site
  • Ampla gama de propriedades para estilizar elementos

Limitações

  • Pode ser desafiador para iniciantes compreender totalmente os conceitos mais avançados
  • Grande complexidade em projetos de grande escala, especialmente se combinado com frameworks e pré-processadores
  • Alguns recursos CSS podem não ser suportados uniformemente em todos os navegadores

Angular.js

AngularJS é um framework JavaScript de código aberto mantido pelo Google. Ele é usado para construir aplicativos web de página única (SPA), facilitando a criação de interfaces de usuário dinâmicas e interativas.

Prós

  • Facilita o desenvolvimento de SPAs com um modelo de desenvolvimento baseado em componentes
  • Vinculação de dados bidirecional, permitindo atualizações automáticas da interface do usuário quando os dados do modelo são alterados
  • Rico conjunto de recursos, incluindo roteamento, injeção de dependência e testabilidade

Limitações

  • Possível íngreme curva de aprendizado devido à complexidade do framework
  • A transição para versões mais recentes do AngularJS pode exigir alterações significativas no código devido a atualizações incompatíveis
  • A documentação oficial pode ser extensa e, às vezes, difícil de navegar

React.js

React.js é uma biblioteca JavaScript de código aberto usada para criar interfaces de usuário (UI) em páginas web. Desenvolvida pelo Facebook, é especialmente eficaz para construir aplicativos de página única (SPAs) e componentes reutilizáveis.

Prós

  • Facilita a criação de interfaces de usuário altamente dinâmicas e interativas
  • Abordagem baseada em componentes, o que permite a reutilização de código e uma estrutura organizada
  • Alto desempenho devido ao seu Virtual DOM, que minimiza as atualizações desnecessárias na interface do usuário

Limitações

  • Exige conhecimento de JavaScript e JSX para utilização eficaz
  • Possível íngreme curva de aprendizado
  • Como é uma biblioteca e não um framework completo, pode ser necessário integrar outras bibliotecas para funcionalidades específicas

Projetos que utilizam:

Facebook Instagram WhatsApp Web Netflix Airbnb Uber

Flutter

Flutter é um framework de código aberto desenvolvido pelo Google para criar aplicativos nativos para dispositivos móveis, web e desktop a partir de uma única base de código. Ele utiliza a linguagem de programação Dart e oferece um conjunto de widgets altamente personalizáveis para criar interfaces de usuário bonitas e fluidas.

Prós

  • Desenvolvimento de aplicativos multiplataforma utilizando um único código
  • Alta performance devido à sua arquitetura baseada em widgets
  • Interface de usuário altamente personalizável, com flexibilidade no design

Limitações

  • Necessidade de aprender Dart
  • Tamanho do aplicativo pode ser maior devido à inclusão do Flutter Engine
  • Alguns recursos nativos podem não estar totalmente suportados ou requerer plugins de terceiros

Projetos que utilizam:

Google Ads Alibaba Group Tencent BMW Nubank Realtor.com

Dart

Dart é uma linguagem de programação desenvolvida pelo Google, projetada para criar aplicativos multiplataforma, especialmente para web e mobile. Ela é conhecida por sua sintaxe limpa e fácil de aprender, bem como por sua performance e eficiência.

Prós

  • Sintaxe simples e familiar, semelhante a outras linguagens como Java e JavaScript
  • Suporte para programação assíncrona
  • Ferramentas poderosas, incluindo um compilador just-in-time (JIT) e um compilador ahead-of-time (AOT)

Limitações

  • Comunidade menor em comparação com outras linguagens mais estabelecidas
  • Menos comum fora do ecossistema de Flutter
  • Recursos e bibliotecas da linguagem ainda podem estar em desenvolvimento

Projetos que utilizam:

Google Ads Alibaba Group Tencent BMW Nubank Realtor.com

MongoDB

MongoDB é um banco de dados NoSQL, orientado a documentos, que oferece alta performance, escalabilidade e flexibilidade. Ele armazena dados em documentos JSON (BSON, para ser preciso) e é amplamente utilizado em aplicações web modernas.

Prós

  • Modelo de dados flexível
  • Escalabilidade horizontal fácil de implementar
  • Consultas poderosas e indexação eficiente

Limitações

  • Menor maturidade em comparação com bancos de dados relacionais
  • Consistência eventual em vez de consistência forte
  • Necessidade de compreender bem o modelo de dados orientado a documentos

Adobe Verizon Bosch eBay The Weather Channel City of Chicago

MySQL

MySQL é um sistema de gerenciamento de banco de dados relacional de código aberto, desenvolvido pela Oracle Corporation. Ele é amplamente utilizado em aplicativos web e outras aplicações que requerem armazenamento e gerenciamento eficiente de dados.

Prós

  • Confiabilidade e estabilidade
  • Amplamente suportado pela comunidade e pela indústria, com vários recursos e ferramentas
  • Facilidade de uso e administração, com uma sintaxe SQL intuitiva e interface amigável

Limitações

  • Escalabilidade horizontal limitada em comparação com sistemas NoSQL
  • Gerenciamento de esquema pode ser complexo em ambientes com muitas alterações de estrutura de dados
  • Limitação em recursos avançados de análise e processamento de dados
